## 5. 개인정보·보안(Data safety)

| 항목 | 예시 답변 |
|------|-----------|
| 사용자 데이터를 수집하나요? | **예** (Google 로그인 사용 시) / 게스트만이면 수집 최소 |
| 수집 목적 | 앱 기능(계정 식별), (선택) 진행 동기화 |
| 수집 유형 | **계정 정보**(Google 계정 ID 등), 동의 범위 내 이름/이메일(요청 시에만) |
| 기기 또는 기타 | 게스트: 로컬 세이브만 → “앱에서만 처리/수집 안 함”에 가깝게 설정 |
| 데이터 공유 | **제3자와 판매하지 않음**. Google 인증 처리는 Google 정책 따름 |
| 암호화 전송 | 서버 통신 시 **예**(HTTPS). 로컬만이면 해당 항목에 맞게 |
| 계정 삭제 | Google 연동 해지·문의 메일로 삭제 요청 가능하다고 안내 |
| 개인정보처리방침 URL | `https://【도메인】/privacy.html` |

**게스트만 먼저 올리고 Google 로그인을 아직 안 넣었다면:**  
Data safety는 “계정 정보 수집 없음”, 앱 액세스는 **아니요**로 맞출 수 있습니다. Google 로그인 넣는 시점에 위 표로 바꾸세요.

## 6. 앱 콘텐츠 / 등급(설문)

| 문항 유형 | 예시 |
|-----------|------|
| 카테고리 | **게임 > 캐주얼** 또는 **전략** |
| 폭력 | 환상적/카툰 스타일 전투(몬스터 처치) — **경미한 환상 폭력** 수준으로 답변 |
| 성적 콘텐츠 | **없음** |
| 언어(욕설) | **없음** |
| 통제 불가능한 사용자 소통 | **없음** |
| 도박 | **없음** |
| 공포/공포물 | 보스 등 약간 있을 수 있음 → 설문 문구에 맞게 **없거나 경미** |
| 타깃 연령 | 보통 **만 3세+ ~ 12세+** 범위에서 설문 결과에 따름 (환상 전투면 PEGI/IARC가 자동 산출) |

설문은 IARC가 등급을 뽑으므로, **실제 앱에 없는 항목은 전부 아니요**가 안전합니다.

## 12. 비공개/내부 테스트 체크리스트

1. 개인정보처리방침 FTP 업로드 + URL 연결  
2. 앱 액세스 **예** + 게스트/테스트 계정 안내 + 전체 액세스 확인 체크  
3. 광고·IAP·UGC·유저 상호작용 → 현재 **아니요**  
4. Data safety를 Google 로그인 여부에 맞게 작성  
5. 콘텐츠 등급 설문 완료  
6. 테스트 트랙에 테스터 이메일/그룹 추가 후 AAB/APK 업로드
